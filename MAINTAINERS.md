# Maintainers

CleanCloud is maintained by a small group of contributors who help review changes, triage issues, and ensure the project remains stable, secure, and useful.

Maintainers are responsible for:

- Reviewing and merging pull requests
- Helping with releases and versioning
- Reviewing issues and discussions
- Maintaining CI / security checks

Maintainers do not gain ownership of the project or any associated intellectual property.

## Becoming a Maintainer

New maintainers may be added to help with:

- Code review
- Testing and validation
- Documentation
- Security review

Maintainers are expected to:

- Act in good faith
- Follow the project's contribution guidelines
- Respect that CleanCloud is a read-only, safety-first cloud hygiene tool

If you're interested in contributing more actively, please open an issue or reach out via GitHub Discussions.

## Contributions & Intellectual Property

By contributing to this repository:

- You agree that your contributions may be included in the open-source project
- Contributions are provided under the project's open-source license
- No additional rights, ownership, or compensation are implied

Contributors are encouraged to ensure they are permitted by their employer to contribute to open-source projects in their own time.

## Security & Responsible Use

CleanCloud is designed to be:

- Read-only
- Deterministic
- Safe to run in sandbox or non-production environments

Maintainers should preserve these principles when reviewing or approving changes.
